如何建立固定排程执行特定行为，本案例会使用 whenever 这个 gem 、每天早上十点固定发送邮件的情况为案例，参考资料：

https://github.com/javan/whenever

1、whenever gem 与 corntab 介绍 ​

crontab 命令常见于 “Unix” 和 “类Unix” 的操作系统之中，用于设置周期性被执行的指令，该命令从标准输入设备读取指令，并将其存放于 ”crontab“ 文件中，以供之后读取和执行。该词来源于希腊语 chronos（χρόνος），原意思是时间。 通常 crontab 储存的指令被守护进程激活，crond 常常在后台运行，每一分钟检查是否有预定的作业需要执行，这类作业一般称为 cron jobs。也就是说 crontab 是在 ”unix“ 和 ”类unix“ 系统中用来实现周期性功能的指令，在网上搜一下，我们就会看到很多 crontab 指令相关的语法

根据上述的分析，我们可以得出这样的结论：

  • whenever 事实上是一个 cron 翻译器，它将 rails 中的 ruby 代码翻译成 cron 脚本，从而将周期性的任务交给 cron 来执行。
  • 这样通过 whenever 我们可以使用 ruby 语言来写周期性任务代码，在 ruby 层控制代码，而不需要与 shell 脚本进行切换；
  • 另一方面，我们会发现，由于 cron 命令的强大，它的语法也因此变得很复杂，通过 whenever，我们可以很方便的实现周期性任务。

2、安装 whenever ​

首先安装 whenever

gem 'whenever', require: false

接着我们使用 wheneverize 以下的指令建立 config/schedule.rb

$ cd /apps/my-great-project
$ bundle exec wheneverize .

这个过程创建了 config 目录 和 schedule.rb 文件

schedule.rb 这个文件可以让我们设定固定排程，我们稍晚会介绍更多如何使用 schedule.rb

3、whenever 常用指令 ​

whenever 是十分亲民的，它的指令没有 cron 命令那么复杂。下面提供几种 whenever 比较常用的方法：

ruby
whenever （不带参数）将schedule.rb 文件中的周期性任务转化成cron命令显示出来，但不读取或者写入到crontab文件中
whenever -i [identifier] 更新新定时任务，默认更新 schedule.rb 文件中的全部
whenever -w, --write-crontab， --update-crontab， [identifier] 写定时任务，默认更新 schedule.rb 文件中的全部
whenever -c, --clear-crontab [identifier] 清除 crontab

除此之外，cron 命令也是可以是用的，比较常用的是：

ruby
crontab -l [UserName]: 列出目前的时程表
crontab -e [UserName]: 执行文字编辑器（第一次由自己选择编辑器）来配置crontab的具体执行方法

4、whenever job_type 排程类型 ​

whenever 预设接受以下四种 job_type，command、rake、runner 接下来我们会聚焦中使用 rake 来帮助我们撰写排程。

ruby
every 3.hours do 
	# 1.minute 1.day 1.week 1.month 1.year is also supported 
	runner "MyModel.some_process" 
	rake "my:rake:task" 
	command "/usr/bin/my_great_command"
end

every 1.day, :at => '4:30 am' do 
	runner "MyModel.task_to_run_at_four_thirty_in_the_morning"
end

every :hour do 
	# Many shortcuts available: :hour, :day, :month, :year, :reboot 
	runner "SomeModel.ladeeda"
end

every :sunday, :at => '12pm' do 
	# Use any day of the week or :weekend, :weekday 
	runner "Task.do_something_great"
end

every '0 0 27-31 * *' do 
	command "echo 'you can use raw cron syntax too'"
end

# run this task only on servers with the :app role in Capistrano
# see Capistrano roles section below
every :day, :at => '12:20am', :roles => [:app] do 
	rake "app_server:task"
end

4.1 rake job 发送邮件案例 ​

建立 Mailer ​

在 app/mailer 下建立一个 daily_mailer.rb 的文件，接着我们定义一个 send_notification method，而我们预期当我們触发这个方法时，我们能够发送 email 给所有的使用者。

ruby
class DailyMailer < ApplicationMailer
  def  send_notification
    default to: -> { User.pluck(:email) },
    default :from "info@example.com"

    mail(
      subject: "「電子報」"
    )
  end
end

建立 rake 指令 ​

首先在 lib/tasks 下建立一个 daily_routine.rake，接着建立一个 task

ruby
require 'rake'

# 用于发送每日邮件的任务
desc "A task used for sending daily mail"

task :send_daily_mail do
  DailyMailer.send_notification.deliver_later!
end

当我们执行 bundle exec rake send_daily_mail RAILS_ENV=your_environment 就可以触发这个 task

设定 schedule.rb ​

接着让我们来设定我们的 schedule.rb

因为我们预期是每天的早上十点会触发固定邮件发送的这个行为，所以我们在 schedule.rb 里做以下的设定

ruby
set :output, "log/cron_log.log"          # 这段指令可以让我们把 log 存在 log/cron_log.log 这个文件中

env :PATH, ENV['PATH']                   # 这段指令确保你可以载入正确的路径

set :environment, :development           # whenever 预设的环境是 production，如果你有想要指定的环境，可以透过这个指令设定

every 1.day, at: '10:00' do
  rake "send_daily_mail"
end

4.2 自定义 job_type ​

在执行定时器任务的时候，使用 rbenv 插件管理 ruby 版本环境来执行 rake，所以我们这里自定义了 rbenv_rake 类型的 job

ruby
# 设置输出日志的文件
set :output, '/home/ubuntu/SaturnCron/log/cron.log'

# 自定义的 job_type 使用了 rbenv，%Q 类似于双引号
# 结构命令：export PATH 将 ”/home/ubuntu/bin“ 、”/home/ubuntu/.rbenv/shims“、”/home/ubuntu/.rbenv/bin“、”/usr/bin:$PATH“ 导入到环境变量
#         eval "$(rbenv init -)" 初始化 rbenv
#         cd :path 进入到项目目录，如果我们在项目下执行 whenever 生成 crontab，这样 path 默认就代表着项目目录
#         RAILS_ENV=production bundle exec rake :task --silent 执行 rake 
#         :output 输出日志
job_type :rbenv_rake, 
				 %Q{export PATH=/home/ubuntu/bin:/home/ubuntu/.rbenv/shims:/home/ubuntu/.rbenv/bin:/usr/bin:$PATH; eval "$(rbenv init -)"; \
            cd :path && RAILS_ENV=production bundle exec rake :task --silent :output }
# DOTO: :task 是变量占位插槽

# 编写任务1
every '5 * * * *' do
  # 上面定义了 :task 变量占位插槽，那 'cron:yesteryear_moments' 默认就是对应着 :task
  # 我们也可以 rbenv_rake rake: 'cron:yesteryear_moments' 如果上面还定义了 :code 插槽，可以这样 rbenv_rake rake: 'xxxx', code: 'xxx'
  # 作者解释: task 总是替换为第一个参数，以及任何附加的 :whatevers 都替换为传入的选项或用 set 定义的变量。
  rbenv_rake 'cron:yesteryear_moments'
end

# 编写任务2
every '5 * * * *' do
  # 如果你想在同一个文件中指定，不相同的项目使用不同的 corn，可以自定义 :path 和日志目录 output
  rbenv_rake 'cron:birthday', path: '/path/to/your/app', output: '/home/ubuntu/SaturnCron/log/cron.log'
end

5、开始执行周期性任务 ​

接着我们要开始执行我们的固定排程

在 console 执行以下的指令

$ whenever -i
=> [write] crontab file updated

这个指令会更新 crontab ，这样我们的排程就可以开始执行了

$ whenever -w
[write] crontab file written

你也可以通过以下的指令列出目前所有的排程，到这里基本上整个流程就结束了。

$ crontab -l

6、部署 ​

如果你的正式机部署是搭配 capistrano 的话，在 Capfile 加入以下一行

ruby
# 使用 bundler 的时候加上这行
set :whenever_command, "bundle exec whenever"
 
require "whenever/capistrano"

這樣可以部署時會執行whenever:update_crontab，將設定新增到server上

7、疑难排解 ​

如果你的排程沒有正常执行，你可以到 log/cron_log.log 来检查是否有错误信息。
