2.2 流程控制 switch-case ​

switch-case 是流程控制用法，和其他语言一样大同小异，只是有些使用习惯上有一些不一样，这里进行逐个介绍 ！

go
package main

import "fmt";

func main(){
	/**
	 * @description 第一种用法是普通的用法
	 * @time 2021-02-16
	 * @input login sucess ！
	 */
	var username string = "zouzhuQcom@163.com";
    
	switch (username) {
		case "zouzhuQcom@163.com":
			fmt.Println("login sucess ！");
		case "hello word":
			fmt.Println("hello word user ！");
		default:
			fmt.Println("none run default ！")

	}
}

switch 后面可以接收一个函数，只要 case 值，符合函数的返回值就可以继续执行后面的代码。

go
package main

import "fmt";

func main(){
	/**
	 * @description 第二种用法，它可以接收一个函数，只要符合函数的返回值即可
	 * @time 2021-02-16
	 * @input this is 3 ！
	 */
	switch performance(61) {
		case 1:
			fmt.Println("this is 1 ！");
		case 2:
			fmt.Println("this is 2 ！");
		case 3:
			fmt.Println("this is 3 ！");
		default:
			fmt.Println("this is 0 ！");

	}
}

/**
 * @description 模拟成绩判断优良差学生
 * @time 2021-02-16
 */
func performance(number float32) int32{
	if number > 90 {
		return 1;
	} else if number > 80 {
		return 2;
	} else if number > 60 {
		return 3;
	} else {
		return 0;
	}
}

如果 switch 后面不接收任何的变量、表达式、函数，当不接任何东西时 switch - case 就相当于 if - elseif - else 。

go
package main

import "fmt";

func main(){
	/**
	 * @description 第三种用法，switch 后可以不接任何变量、表达式、函数。
	 *				当不接任何东西时 switch - case 就相当于 if - elseif - else
	 * @time 2021-02-16
	 * @input 不合格
	 */
	var score int32 = 30

	switch {
		case score >= 95 && score <= 100:
			fmt.Println("优秀")
		case score >= 80:
			fmt.Println("良好")
		case score >= 60:
			fmt.Println("合格")
		case score >= 0:
			fmt.Println("不合格")
		default:
			fmt.Println("输入有误...")
	}
}

正常情况下 switch - case 的执行顺序是，只要有一个 case 满足条件，就会直接退出 switch - case ，如果一个都没有满足，才会执行 default 的代码块。 但是有一种情况是例外，那就是当 case 使用关键字 fallthrough 开启穿透能力的时候。需要注意的是，fallthrough 只能穿透一层，意思是它让你直接执行下一个case的语句，而且不需要判断条件。

go
var message string = "hello";

switch {
	case message == "hello":
    	fmt.Println("hello")
    	fallthrough
	case message == "xxxx":
    	fmt.Println("xxxx")
	case message != "world":
    	fmt.Println("world")
}

>> hello
>> xxxx

DANGER

case 后可以接多个多个条件，多个条件之间是 || 或的关系，用逗号相隔，且全部 case 值都是不能重复的，不然程序就会报错，下面就是错误的代码示范 ！

go
var gender string = "male"

switch gender {
    case "male":
        fmt.Println("男性")
    case "male":					// 1.与上面重复
        fmt.Println("男性")
    case "female":
        fmt.Println("女性")
}

switch gender {
    case "male", "male":		   // 2.左边跟右边重复
        fmt.Println("男性")
    case "female":
        fmt.Println("女性")
}

2.3 流程控制 for 循环 ​

for 是流程控制用法，和其他语言一样大同小异，只是有些使用习惯上有一些不一样，这里进行逐个介绍 ！

go
var a number = 1;

for a <= 5 {
    fmt.Println(a)
    a ++
}

for i := 1; i <= 5; i++ {
    fmt.Println(i)
}

在 Go 语言中，没有 while 循环，如果要实现无限循环，也完全可以 for 来实现 。当你不加任何的判断条件时，就相当于你每次的判断都为 true，程序就会一直处于运行状态，但是一般我们并不会让程序处于死循环，在满足一定的条件下，可以使用关键字 break 退出循环体，也可以使用 continue 直接跳到下一循环 。

go
for {
    fmt.Println("开启无限循环 ！");
    break;
}

遍历一个可迭代对象，是一个很常用的操作，在 Go 可以使用 for-range 的方式来实现，range 后可接数组、切片，字符串等；由于 range 会返回两个值：索引和数据，若你后面的代码用不到索引，需要使用 _ 表示 。 如果你用一个变量来接收的话，则接收到的是索引 。

go
import "fmt"

func main() {
    myarr := [...]string{"world", "python", "go"}
    
    for _, item := range myarr {
        fmt.Printf("hello, %s\n", item)
    }
}


>> hello, world
>> hello, python
>> hello, go

2.4 流程控制 goto 无条件跳转 ​

goto 关键字的作用就是无条件跳转，很难想象在 Go 里面居然会保留 goto，因为很多人不建议使用 goto，所以在一些编程语言中甚至取消了 goto，我们先了解它的一个具体使用方式 。

go
package main

import "fmt";

func main(){
	/**
	 * @description goto 可以打破原有的代码规则，直接跳转到某一行执行代码，这里声明了一个
	 *                   flag 代码块，通过 goto flag 就直接跳到了 flag, 而直接忽略掉
	 *                   中间打印的输出 ！
	 * @time 2021-02-21
	 *
	 * A
	 */
	goto flag;
	fmt.Println("B");
	flag:
		fmt.Println("A");
}

goto 语句通常与条件语句配合使用，可用来实现条件转移，构成循环体等功能，我们这里分别使用 glag 来模拟循环，break，continue 的效果。

go
import "fmt"

func main() {
    /**
	 * @description goto 模拟循环效果，来实现一个打印 1到5 的循环
	 * @time 2021-02-21
	 */
    i := 1
	flag:
    	if i <= 5 {
        	fmt.Println(i)
        	i++
        	goto flag					
    	}
}
go
import "fmt"

func main() {
    /**
	 * @description goto 模拟 break 效果
	 * @time 2021-02-21
	 */
    i := 1
    for {
        if i > 5 {
            goto flag
        }
        fmt.Println(i)
        i++
    }
	flag:
}
go
import "fmt"

func main() {
    /**
	 * @description goto 模拟 continue 效果，打印 1到10 的所有偶数
	 * @time 2021-02-21
	 */
    i := 1
	flag:
    	for i <= 10 {
        	if i%2 == 1 {
            	i++
            	goto flag
        	}
        	fmt.Println(i)
        	i++
    	}
}

DANGER

goto语句与标签之间不能有变量声明，否则编译错误，我们在代码编写的时候要注意这个习惯，避免不规范书写代码而导致程序编译报错问题 。

go
import "fmt"

func main() {
    /**
	 * @description goto flag -> flag 之间声明了一个 say 所以编译报错
	 * @time 2021-02-21
	 */
    fmt.Println("start")
    goto flag
    var say = "hello oldboy"
    fmt.Println(say)
	flag:
    	fmt.Println("end")
}

2.5 流程控制 defer 延迟语句 ​

今天是最后一篇讲控制流程了，内容是 defer 延迟语句，这个在其他编程语言里好像没有见到。应该是属于 Go 语言里的独有的关键字，但即使如此，阅读后这篇文章后，你可以发现 defer 在其他编程语言里的影子，下面就是 defer 的结构，类似于 Java 中的栈结构一样 。

go
package main

import "fmt";

func main(){
	/**
	 * 我们以下面这段代码为例, defer 是延迟函数会先压入类似到栈结构中(后进先出)
	 * defer 会在程序后面的代码完全执行完毕后才依次弹出执行，这样起到了一个延迟的作用，
	 * 使用 defer 只是延时调用函数，此时传递给函数里的变量，不应该受到后续程序的影响。
	 * 像这里虽然改变了 name 的值，后续调用 defer 的时候，仍然使用未重新赋值的变量值，
	 * 就好在 defer 这里，给所有的这是做了一个快照一样。
	 */
	var name string = "go";
	defer fmt.Println(name); 	// 输出: go             defer1

	name = "python";
	defer fmt.Println(name); 	// 输出: python		  defer2

	name = "java";
	fmt.Println(name);
}

上面的图例就是 defer 的结构，就好像 Java 中压栈一样，运行代码后 defer 会陆续的进入一个栈结构，待代码执行完后才会依次弹出 defer 进行执行，遵守着后进先出的原则，只是类似栈，但不是真正的栈结构 。还有一种 defer 的复杂情况，就是当 defer 、return 一起使用的时候，哪一个先执行，哪一个后执行呢 ？看下面这段代码的输出进行分析 。

go
package main

import "fmt";

var name string = "go"

func myfunc() string {
	defer func() {
		name = "python"
	}()

	fmt.Printf("myfunc 函数里的name：%s\n", name)
	return name
}

func main() {
	/*
	 * 第一行很直观，name 此时还是全局变量，值还是 go
	 * 第二行也不难理解，在 defer 里改变了这个全局变量，此时 name 的值已经变成了 python
	 * 重点在第三行，为什么输出的是 go ？解释只有一个，那就是 defer 是 return 后才调用的。
	 * 所以在执行 defer 前，myname 已经被赋值成 go 了。
	 * 所以在程序编写的时候，我们一定要记住 return 在 defer 前面就执行了。
	 */
	myname := myfunc()
	fmt.Printf("main 函数里的name: %s\n", name)
	fmt.Println("main 函数里的myname: ", myname)
}

第 3 章 Go 面向对象编程 ​

Go 语言并不是完全面向对象的，其实 Go 语言中并没有类和对象的概念，首先 Go 语言是完全可以写成面向过程风格的。Go 语言中有很多的 function 函数类型是不属于任何对象的。

Java 语言中，封装是自然而来的，也是强制的。你所写的代码，都要属于某个类，某个class文件。类的属性封装了数据，方法则是对这些数据的操作。通过 private 和 public 来控制数据的可访问性。每个类（java文件），自然的就是一个对象的模板。 然而 Go 中则没有这样的概念，Go 封装有包范围的封装和结构体范围的封装。

3.1 结构体与继承 ​

在之前学过的数据类型中，数组与切片都只能存储同一类型的变量，若要存储多个类型的变量就需要用到结构体，每个变量都成为该结构体的成员变量，可以理解为 Go语言的结构体 struct 和其他语言的 class 有相等的地位，但是 Go 语言放弃大量面向对象的特性，所有的 Go 语言类型除了指针类型外，都可以有自己的方法, 提高了可扩展性。

在 Go 语言中没有没有 class 类的概念，只有 struct 结构体的概念，因此也没有继承，本篇文章将带你学习一下结构体相关的内容。

go
package main

import "fmt"

func main(){
	/*
	 * 下面我们实例化了结构体，并将实例的 person 赋值给了 Company Person 匿名字段，这样我们
	 * 就可以通过 company 实例调用 person 里面的属性了，用结构体中的组合，模拟出了类继承的效果
	 */
	person := Person{
		personName: "邹柱",
		personDescribed: "前端开发",
	}
	company := Company{
		companyName: "上海高企集团有限公司",
		companyDescribed: "主要从事汽车服务咨询的业务",
		Person: person,
	}
	company.printCompanyName()
	company.changeCompanyName("蓝色愿景集团有限公司")
	company.printCompanyName()
}

/*
 * Go 语言中本身是不支持继承的，也没有继承这个功能，但我们可以使用组合的方法，实现类似继承的效果。
 * 在生活中组合的例子非常多，比如一台电脑是由机身外壳、主板、CPU、内存等零部件组合在一起，最后才
 * 有了我们用的电脑，同样的在 Go 语言中，把一个结构体嵌入到另一个结构体的方法，称之为组合。
 *
 * 下面我们定义了一个 Company 公司的结构，还定义了一个 Person 人员的结构，若要将公司信息与公司
 * 人员关联起来，我们就可以使用组合，可以将 Person 这个结构体镶嵌到 Company 中，作为 Company
 * 的一个匿名字段，这样 Company 就直接拥有了 Person 的所有属性了
 */
type Company struct {
	companyName string
	companyDescribed string
	Person
}

type Person struct {
	personName string
	personDescribed string
}

/*
 * 结构体内部是无法定义方法的，那如何给一个结构体定义方法呢 ？可以使用组合函数的方式来定义结构体方法，
 * 它和普通函数的定义方式有些不一样，其中 printCompanyName 是方法名，而 (company Company) 表
 * 示将 printCompanyName 方法与 Company 的实例绑定，我们把 Company 称为方法的接收者，而 company
 * 表示实例本身。
 *
 * 组合结构后，Company 就拥有了 Person 的全部属性和方法，我们可以通过 Person 字段调用，或者是直接调用。
 */
func(company Company) printCompanyName(){
	fmt.Printf("公司的名称 : %s\n", company.companyName)
	fmt.Printf("公司的描述 : %s\n", company.companyDescribed)
	fmt.Printf("公司的人员 : %s - %s\n", company.personName, company.Person.personName)
}

/*
 * 当你想要在方法内改变实例的属性的时候，必须使用指针作为方法的接收者，才能改变属性。
 */
func(company *Company) changeCompanyName(companyName string){
	company.companyName = companyName
}

在 Go 语言中，函数名的首字母大小写非常重要，它被来实现控制对方法的访问权限。

  • 当方法的首字母为大写时，这个方法对于所有包都是Public，其他包可以随意调用。
  • 当方法的首字母为小写时，这个方法是Private，其他包是无法访问的。

3.2 接口与多态 ​

在面向对象的领域里，接口可以定义一个对象的行为，接口只指定了对象应该做什么，至于如何实现这个行为(既实现细节)，则由对象本身去确定，在 Go 语言中，接口就是方法签名的集合，当一个类型定义了接口中的所有方法，我们称它实现了该接口，这与面向对象编程 (OOP) 的说法很类似，接口指定了一个类型应该具有的方法，并由该类型决定如何实现这些方法 。

go
package main

func main() {
	/*
	 * 因为实现是隐式的，这里还没有办法论证是否实现了 Phone 接口，所以我们把 xiaomi
	 * 的类型声明为 Phone 接口结构类型，如果此时实现了接口里面的方法则不会报错，如果
	 * 没有完全实现接口方法，则 IDE 会报错.
	 */
	var xiaomi Phone = XiaoMi{
		description: "我是今年最新款小米手机 ！",
	}
	xiaomi.call()
}

/*
 * 我们这里定义了一个手机的接口类型，手机都要实现能够 call 打电话 game 游戏功能，
 * 只要实例实现了接口里面的方法，那它就是 Phone 接口结构对象。
 *
 * Go 不像 Java 语法那样有 implement interface 关键字去实现接口，Go 接口的实现
 * 是隐式的，只要实现了接口里面的方法，就认为是 Phone 接口类型的实现.
 */
type Phone interface {
	call() string
	game() string
}

type XiaoMi struct {
	description string
}

func (xiaomi XiaoMi) game() string {
	return "喂 ！你好，我能玩王者荣耀 ！"
}

func(xiaomi XiaoMi) call() string{
	return "喂 ！你好，我是小米手机 ！"
}

多态：我们定义一个商品接口的形态，只要实现了接口里面的方法我就认为你是一个商品，至于商品的具体实现统统不管，这就是接口的多态性 (这个商品可以是商场手机或者是商场赠品) ，我们这里利用接口的多态性实现一个商场收银结算功能 。

go
package main

import (
	"fmt"
	"strconv"
)

// 商品类型接口
type Good interface {
	settleAccount() int      // 计算商品的总价
	orderInfo() string		 // 打印商品发票
}

// 手机
type Phone struct {
	name string		 // 手机名称
	quantity int     // 手机数量
	price int		 // 手机价格
}

func (phone Phone) settleAccount() int {
	return phone.quantity * phone.price
}

func (phone Phone) orderInfo() string{
	return "您要购买" + strconv.Itoa(phone.quantity)+ "个" +
		phone.name + "计：" + strconv.Itoa(phone.settleAccount()) + "元"
}

// 耳机
type FreeGift struct {
	name string		// 耳机名称
	quantity int	// 耳机数量
	price int		// 耳机价格
}

func (gift FreeGift) settleAccount() int {
	return 0
}

func (gift FreeGift) orderInfo() string{
	return "您要购买" + strconv.Itoa(gift.quantity)+ "个" +
		gift.name + "计：" + strconv.Itoa(gift.settleAccount()) + "元"
}

// 收银台计算商品总价
func calculateAllPrice(goods []Good) int {
	var allPrice int
	for _,good := range goods{
		fmt.Println(good.orderInfo())
		allPrice += good.settleAccount()
	}
	return allPrice
}

func main() {
	/*
	 * 这里分别实例化了手机跟耳机，然后将它们传递给 Good 类型的数组，这就是需要结账的所有
	 * 商品，将 goods 传递给 calculateAllPrice 收银台方法进行结算和打印发票，这就是
	 * 多态的应用了
	 */
	iPhone := Phone{
		name:     "iPhone",
		quantity: 1,
		price:    8000,
	}
	earphones := FreeGift{
		name:     "耳机",
		quantity: 1,
		price:    200,
	}

	goods := []Good{iPhone, earphones}
	allPrice := calculateAllPrice(goods)
	fmt.Printf("该订单总共需要支付 %d 元", allPrice)
}

4.1 进程、线程、协程的概念 ​

进程是什么呢？

直白地说进程就是应用程序的启动实例，比如我们运行一个游戏，打开一个软件，就是开启了一个进程。进程拥有代码和打开的文件资源、数据资源、独立的内存空间 。

线程又是什么呢？

线程从属于进程，是程序的实际执行者，一个进程至少包含一个主线程，也可以有更多的子线程。

线程拥有自己的栈空间。

进程、线程总结 ？

有人给出了很好的归纳，对于操作系统来说，线程是最小的执行单元，进程是最小的资源管理单元，无论进程还是线程，都是由操作系统所管理的。

那进程和线程的痛点是什么呢 ？线程之间是如何进行协作的呢 ？最经典的例子就是生产者/消费者模式：

若干个生产者线程向队列中写入数据，若干个消费者线程从队列中消费数据。

  1. 定义了一个生产者类，一个消费者类。
  2. 生产者类循环100次，向同步队列当中插入数据。
  3. 消费者循环监听同步队列，当队列有数据时拉取数据。
  4. 如果队列满了（达到5个元素），生产者阻塞。
  5. 如果队列空了，消费者阻塞。

上面的流程正确地实现了生产者/消费者模式，但是却并不是一个高性能的实现。为什么性能不高呢？以下涉及到的任何一点，都是非常耗费性能的操作 ：

  1. 涉及到同步锁。
  2. 涉及到线程阻塞状态和可运行状态之间的切换。
  3. 涉及到线程上下文的切换。

什么是协程 ？

协程，英文Coroutines，是一种比线程更加轻量级的存在。正如一个进程可以拥有多个线程一样，一个线程也可以拥有多个协程，最重要的是协程不是被操作系统内核所管理，而完全是由程序所控制（也就是在用户态执行）。这样带来的好处就是性能得到了很大的提升，不会像线程切换那样消耗资源，既然协程这么好，到底好在哪里呢 ？

我们现在以生成一个协程和生成一个线程操作为例，创建一个线程需要消耗系统资源，而创建一个协程完全靠用户决定就好了，现在让线程暂停，我们需要先手动暂停线程，然后系统才发现这个线程没有工作了，然后才能暂停这个线程，然后去安排可运行的新线程去工作 。而从生成一个协程，然后让协程暂停，其中协程和线程的阻塞是有本质区别的，协程的暂停完全由程序控制，线程的阻塞状态是由操作系统内核来进行切换，因此协程的开销远远小于线程的开销 。

4.2 协程的初步了解与使用 goroutine ​

go
package main

import (
	"fmt"
	"time"
)
/*
 * go 关键字就是语言中生成一个协程的方式，一个 goroutine 本身就是一个函数，当你直接调用时，
 * 它就是一个普通函数，如果你在调用前加一个关键字 go ，那你就开启了一个 goroutine 协程。
 * main goroutine 相当于主线程，当主线程执行完后，它是不会继续等待协程执行完成的，所以为了
 * 更加直观的查看协程的效果，我们使用 time.Sleep 的方式堵塞主线程 1 秒.这并不是推荐的方式，
 * 后面我们将介绍更加优雅的方式 ！
 *
 * 我们这里 Go 生成了两个协程 goroutine，分别负责去生产苹果和葡萄，可以观察到两个协程就如两
 * 个线程一样，并发执行我们的生产任务了。
 *
 * 通过以上简单的例子，是不是折服于 Go 的这种强大的并发特性，将同步代码转为异步代码，真的只要
 * 一个关键字就可以了，也不需要使用其他库，简单方便。
 */
func main(){
	go productionPingGuo()
	go productionPuTao()
	time.Sleep(1000);
	fmt.Println("主进程结束 !")
}

func productionPingGuo(){
	for i := 1; i<=10; i++ {
		fmt.Println("production 🍎")
	}
}

func productionPuTao(){
	for i := 1; i<=10; i++ {
		fmt.Println("production 🍇")
	}
}

4.3 协程数据通信(信道-通道) ​

Go 语言之所以开始流行起来，很大一部分原因是因为它自带的并发机制，如果说 goroutine 是 Go 语言程序的并发体的话，那么 channel（信道） 就是 它们之间的通信机制 ，channel 是一个可以让一个 goroutine 与另一个 goroutine 传输信息的通道，我把他叫做信道，也有人将其翻译成通道，二者都是一个概念。

信道就是一个管道，连接多个 goroutine 程序 ，它是一种队列式的数据结构(例如消息队列结构)，遵循先入先出的规则。
